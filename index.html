How it works

Five steps, start to finish

Every message — the ones used to train the model, and the one you type into the demo — passes through the exact same pipeline.

01

Clean

Lowercase the text and strip punctuation, numbers, and symbols.

02

Filter

Remove common filler words ("the", "is", "at") that carry no signal.

03

Vectorize

Count how often each remaining word appears — a "bag of words".

04

Train

Naive Bayes learns the odds of each word appearing in spam vs. genuine messages.

05

Predict

New text is scored against both odds, and the higher probability wins.

Results

We tested three algorithms first

Same cleaned data, same word features, three different classifiers — compared before picking a winner.

Naive Bayes97.4% accuracy · 93.0% precision
Logistic Regression97.6% accuracy · 98.3% precision
SVM (Linear)97.1% accuracy · 95.8% precision

Logistic Regression edged ahead on precision, but Naive Bayes was chosen: it's built directly on Bayes' theorem — simple to explain, fast enough to run instantly in a browser with no server, and still catches spam with high accuracy.
